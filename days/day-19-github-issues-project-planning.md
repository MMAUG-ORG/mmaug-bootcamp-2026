# Day 19: GitHub Issues and Project Planning

## Objective

Use GitHub issues to plan capstone work.

## Core Topics

- Issues
- Labels
- Tasks
- Acceptance criteria
- Small work items
- Blockers

## Lesson Content

Planning work in small pieces makes projects easier to finish. Good issues describe what needs to happen and how to know when it is done.

## Hands-On Lab

1. Create three GitHub issues for your capstone.
2. Add acceptance criteria to each issue.
3. Add labels.
4. Link the issues in your notes.

## Deliverable

- `notes/day-19-project-plan.md`

## Mentor Notes

Discourage huge issues like "Build everything." Help learners split tasks.
