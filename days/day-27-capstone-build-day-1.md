# Day 27: Capstone Build Day 1

## Objective

Build the first working version of the capstone.

## Core Topics

- Starting with the smallest useful feature
- Frequent commits
- Basic project structure
- Running locally
- Keeping notes while building

## Lesson Content

The first working version should prove the core idea. It can be simple and imperfect. Once something works, learners can improve it with documentation, validation, automation, and review.

## Hands-On Lab

1. Create the capstone project files.
2. Implement the core flow.
3. Run it locally.
4. Commit the working baseline.
5. Note what remains.

## Deliverable

- First working capstone version

## Mentor Notes

Watch for learners getting stuck on optional features too early.
