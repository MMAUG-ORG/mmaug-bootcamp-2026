# Day 29: Add DevOps, DevSecOps, and AIOps Discipline

## Objective

Add automation, security thinking, and operations thinking to the capstone.

## Core Topics

- CI workflow
- Repeatable commands
- Validation
- Secret checks
- Dependency awareness
- Logs and troubleshooting
- Supportability

## Lesson Content

A capstone should show more than code. It should show delivery thinking. DevOps adds repeatable checks. DevSecOps adds basic security hygiene. AIOps adds awareness of logs, alerts, incidents, and troubleshooting support.

## Hands-On Lab

1. Confirm the GitHub Actions workflow runs.
2. Add a validation or manual testing checklist.
3. Review the repo for secrets or sensitive data.
4. Add a security section to the README.
5. Add an operations or troubleshooting section to the README.

## Deliverable

- Passing workflow or documented validation result
- Security and operations sections in the capstone README

## Mentor Notes

This is where the bootcamp ties the technical habits together.
