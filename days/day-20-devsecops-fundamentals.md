# Day 20: DevSecOps Fundamentals

## Objective

Understand how security becomes part of everyday software delivery.

## Core Topics

- DevSecOps
- Shift-left security
- Secrets and credentials
- Dependency awareness
- Secure code review
- Least privilege

## Lesson Content

DevSecOps means bringing security into the development and operations workflow instead of treating it as a final separate step. For beginners, this starts with simple habits: do not commit secrets, review dependencies, validate input, document risks, and ask what could go wrong.

## Hands-On Lab

1. Create a security checklist for your repo.
2. Check your files for passwords, tokens, private keys, or sensitive data.
3. Add a `.gitignore` note if needed.
4. Write one paragraph explaining least privilege.
