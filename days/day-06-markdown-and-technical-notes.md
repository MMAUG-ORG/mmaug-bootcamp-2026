# Day 6: Markdown and Technical Notes

## Objective

Write clear technical notes using Markdown.

## Core Topics

- Headings
- Lists
- Links
- Code blocks
- Tables
- Daily learning logs

## Lesson Content

Documentation is part of technical work. Markdown is widely used in GitHub READMEs, issue descriptions, project notes, and documentation sites. Good notes help learners remember what they did and help mentors give better support.

## Hands-On Lab

1. Format your Day 1 to Day 5 notes.
2. Add code blocks for commands.
3. Add links to useful resources.
4. Add a progress table to your README.

## Deliverable

- Updated `README.md`

## Mentor Notes

Encourage short, honest notes over polished essays.
