# Day 28: Capstone Build Day 2

## Objective

Improve the capstone for usability, clarity, and reliability.

## Core Topics

- Error messages
- Input validation
- Clear output
- README usage steps
- Manual testing
- Known limitations

## Lesson Content

A project becomes more useful when others can run it and understand it. Today focuses on improving the project experience, not adding unnecessary complexity.

## Hands-On Lab

1. Add helpful output or validation.
2. Add usage steps to the README.
3. Run manual test steps.
4. Record known limitations.
5. Commit improvements.

## Deliverable

- Improved capstone project

## Mentor Notes

Help learners write instructions as if someone else will run the project tomorrow.
