# Day 4: Git and Version Control

## Objective

Understand why version control matters and make the first Git commit.

## Core Topics

- Repository
- Commit
- Working tree
- Staging area
- Git history
- Commit messages

## Lesson Content

Git tracks changes over time. This helps learners see progress, recover from mistakes, and collaborate. DevOps depends heavily on version control because automation usually starts from a repository.

## Hands-On Lab

1. Run `git init` in your bootcamp folder.
2. Run `git status`.
3. Add your notes and templates.
4. Create your first commit.
5. View the commit history.

## Deliverable

- Local Git repository with at least one commit

## Mentor Notes

Focus on the mental model: Git sees changes, learners stage changes, commits save checkpoints.
