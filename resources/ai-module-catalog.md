# AI Fundamentals Module Catalog

This catalog gives facilitators a clear map for the AI fundamentals track. It is beginner-friendly, AI-901/AI-103-inspired, and practical enough for learners who will use AI tools during projects.

## Module 1: AI, Machine Learning, and Deep Learning

Learners should be able to explain:

- AI as software that performs tasks associated with human intelligence.
- Machine learning as systems that learn patterns from data.
- Deep learning as neural-network-based learning.
- Generative AI as systems that create new text, images, code, audio, or other content.
- The difference between training and inference.

## Module 2: Data and Model Basics

Learners should be able to explain:

- Dataset, label, feature, example, training, validation, and test data.
- Why data quality affects AI quality.
- Bias, overfitting, underfitting, and drift at a beginner level.
- Why AI output should be evaluated, not simply accepted.

## Module 3: Large Language Models

Learners should be able to explain:

- What an LLM is and why it predicts useful language from context.
- Chat messages, system instructions, user prompts, assistant responses, and tool messages.
- Embeddings as numeric representations used for search and similarity.
- Multimodal models that can work with text, images, audio, or video.

## Module 4: Tokens, Context Windows, and Cost

Learners should be able to explain:

- Tokenization as the process of splitting text into pieces a model can process.
- Context window as the maximum amount of input and conversation history a model can consider.
- Input tokens, output tokens, token limits, truncation, latency, and cost.
- Why concise context often improves reliability.

## Module 5: Model Behavior Controls

Learners should be able to explain:

- Temperature for more predictable or more varied output.
- Max output tokens for response length control.
- Stop sequences and structured output.
- Why low temperature is usually better for extraction, classification, and repeatable workflows.
- Why higher temperature may help brainstorming but needs stronger review.

## Module 6: Prompt Engineering

Learners should practice:

- Clear task statements.
- Role and audience framing.
- Context and constraints.
- Examples and few-shot prompts.
- Output formats such as tables, JSON, checklists, and summaries.
- Iteration and verification.

## Module 7: Context Engineering

Learners should practice:

- Selecting relevant notes, files, examples, and retrieved content.
- Removing outdated or conflicting context.
- Summarizing long context safely.
- Separating facts from assumptions.
- Designing retrieval augmented generation at a high level.
- Defining what the model should ignore.

## Module 8: Provider and Model Landscape

Learners should compare:

- Microsoft Foundry and Azure OpenAI for enterprise AI apps and Azure integration.
- OpenAI and ChatGPT for general-purpose AI assistance and API-based apps.
- Anthropic Claude for long-context reasoning and assistant workflows.
- Google Gemini for multimodal and Google ecosystem use cases.
- Open-source and local models for experimentation, privacy, customization, and cost control.

The goal is not to memorize every model. The goal is to choose based on task fit, safety, privacy, cost, latency, quality, and integration needs.

## Module 9: Agentic Engineering

Learners should be able to explain:

- Agents as AI systems that can plan, use tools, and take steps toward a goal.
- Tool calling, memory, planning, reflection, and human approval.
- Guardrails, permissions, and audit trails.
- Failure modes such as wrong tool use, hidden assumptions, loops, stale context, and overconfident action.

## Module 10: Responsible AI and Evaluation

Learners should practice:

- Checking for hallucination, bias, privacy risk, harmful output, security risk, and copyright concerns.
- Writing intended-use and out-of-scope notes.
- Creating small test sets.
- Comparing actual output with expected behavior.
- Recording what was accepted, changed, or rejected.
