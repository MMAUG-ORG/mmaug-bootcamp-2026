# Prizes and Eligibility

MMAUG plans to award **5 Microsoft Certification exam vouchers** to the top 5 eligible participants whose final capstone submissions meet all required criteria.

## Prize

- **Prize type**: Microsoft Certification exam voucher
- **Number of prizes**: 5
- **Award recipients**: Top 5 eligible participants based on final capstone review
- **Limit**: One voucher per winning participant

## Eligibility Conditions

To be eligible for a voucher, a participant must:

- Be registered for the MMAUG AI and DevOps Fundamentals Bootcamp.
- Follow the bootcamp code of conduct.
- Submit the final capstone project by the stated deadline.
- Provide a working or reviewable GitHub repository link.
- Use the official capstone submission template.
- Choose one of the approved capstone scenario tracks.
- Meet all minimum capstone acceptance criteria.
- Include evidence of AI, DevOps, DevSecOps, AIOps, and responsible AI work.
- Be able to explain the submitted project during review or showcase if requested.

## Capstone Criteria Required for Prize Consideration

The final submission must include:

- Clear README
- Selected scenario track
- Microsoft Foundry or Azure AI Services usage, or a clear beginner architecture showing how those services fit
- Source files, prompts, scripts, workflow files, or configuration files
- Basic testing or validation evidence
- GitHub Actions workflow or documented validation checklist
- DevSecOps note showing secrets were not committed
- AIOps note showing troubleshooting or operational thinking
- Responsible AI note
- Final reflection
- Demo outline

## Judging Rubric

| Category | Points |
| --- | ---: |
| Problem clarity and target user | 10 |
| Microsoft Foundry or Azure AI Services alignment | 20 |
| Working prototype or clear reviewable implementation | 20 |
| DevOps evidence | 15 |
| DevSecOps evidence | 10 |
| AIOps evidence | 10 |
| Responsible AI note | 10 |
| Documentation and demo clarity | 5 |
| **Total** | **100** |

## Tie-Breakers

If two or more participants have similar scores, reviewers may use the following tie-breakers:

- Clearer explanation during the demo
- Better documentation quality
- Stronger evidence of iteration through commits or issues
- More complete responsible AI and security notes
- Earlier valid submission time

## Disqualification Conditions

A submission may be disqualified from prize consideration if:

- It is submitted after the deadline without organizer approval.
- The repository cannot be accessed by reviewers.
- The participant cannot explain the work.
- The project includes copied work without attribution.
- The repository contains exposed secrets, private keys, tokens, or sensitive data.
- The submission violates the MMAUG code of conduct.
- The project does not meet the minimum capstone acceptance criteria.

## Voucher Notes

- Voucher availability, exam eligibility, expiry dates, and redemption rules may be subject to Microsoft voucher terms.
- Vouchers are not guaranteed for all participants.
- Vouchers are not transferable unless organizers explicitly allow it.
- Organizers may adjust timelines or review logistics if required, but prize criteria should remain transparent to participants.
