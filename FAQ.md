# MMAUG Bootcamp 2026 FAQ

This FAQ adapts the common hackathon and learning-event structure for the Malta Microsoft AI User Group 30-Day AI and DevOps Fundamentals Bootcamp.

## General

### What is the MMAUG AI and DevOps Fundamentals Bootcamp?

It is a 30-day beginner bootcamp created by the Malta Microsoft AI User Group to help learners build the prerequisite skills needed to enter AI, cloud, and DevOps learning paths.

### Is this a hackathon?

No. It is a guided fundamentals bootcamp. However, the structure prepares learners for future hackathons by teaching GitHub, documentation, project planning, AI concepts, and beginner DevOps workflows.

### Who is it for?

It is for beginners, students, career switchers, community members, and early-career professionals who want to understand the basics of AI and DevOps before attempting more advanced projects.

### Do I need prior coding experience?

No. Prior coding experience is helpful but not required. The bootcamp starts with computer, command-line, GitHub, and Python basics.

### Is the bootcamp only for people in Malta?

The bootcamp is organized for the MMAUG community, but materials can be used by anyone who wants to learn. Live sessions, mentoring, or showcase access may depend on the event format chosen by the organizers.

### What language will be used?

The curriculum is written in English.

## Schedule and Format

### How long is the bootcamp?

The bootcamp runs for 30 days.

### How much time should I spend each day?

Plan for 60 to 120 minutes per day. Some days may take less time, especially if your tools are already installed.

### What happens if I miss a day?

Continue from where you stopped. The goal is learning progress, not perfection. Participants should catch up gradually and avoid rushing through hands-on tasks without understanding them.

### Are there live sessions?

The guide supports either self-paced or mentor-led delivery. MMAUG organizers can add live kickoff, weekly review, office hours, and final showcase sessions.

### Will sessions be recorded?

This depends on the delivery format chosen by the organizers. If recordings are provided, links should be added to the bootcamp repository or event page.

## Tools and Accounts

### What accounts do I need?

You need a GitHub account. A Microsoft account is recommended for Microsoft Learn and optional Azure exploration.

### Do I need an Azure subscription?

No Azure subscription is required for the core beginner curriculum. The bootcamp focuses on fundamentals, concepts, GitHub, Python, and automation. Optional Azure labs can be added separately.

### What software should I install?

Recommended tools:

- Visual Studio Code
- Git
- Python 3
- A modern browser

### Can I use GitHub Codespaces?

Yes, if available. Codespaces can reduce local setup issues, but beginners should still understand the basic ideas of files, terminal commands, Git, and dependencies.

### Can I use AI tools during the bootcamp?

Yes. AI tools may be used for explanation, debugging help, and learning support. Participants must still understand and document their work. Do not submit generated code or text that you cannot explain.

## Learning Content

### What topics are covered?

The bootcamp covers:

- Computer and developer basics
- Command line
- Git and GitHub
- Markdown documentation
- Python basics
- AI fundamentals inspired by AI-901/AI-103-style learning outcomes
- Machine learning and generative AI basics
- LLMs, tokens, context windows, embeddings, and model behavior
- Prompt engineering and context engineering
- Model temperature and output controls
- Open and commercial AI provider families, including Microsoft Foundry/Azure OpenAI, OpenAI/ChatGPT, Anthropic Claude, Google Gemini, open-source models, and local models
- Agentic engineering and AI agents at a beginner level
- Responsible AI
- Cloud and Azure fundamentals
- DevOps concepts
- DevSecOps concepts
- AIOps concepts
- Agentic DevOps concepts
- GitHub Actions
- Prompting and AI-assisted coding
- Capstone project delivery

### Is this an official Microsoft certification course?

No. It is a community bootcamp inspired by Microsoft AI and cloud learning paths, including AI-901/AI-103-style fundamentals. It can help learners prepare for later Microsoft Learn study, but it does not replace official exam preparation materials.

### What is the difference between prompt engineering and context engineering?

Prompt engineering is how you write the instruction. Context engineering is how you choose and manage the information the AI receives, such as notes, files, examples, retrieved documents, tool output, and constraints.

### Will I learn about ChatGPT, Claude, and Gemini?

Yes, at a beginner comparison level. The bootcamp does not promote one provider as always best. Learners compare model families by capability, cost, privacy, latency, safety controls, integration options, and fit for the task.

### Will I learn open-source AI?

Yes, conceptually. Learners will discuss open-source and local model options, why teams use them, and what tradeoffs they bring. The core bootcamp does not require learners to run large local models.

### What is agentic DevOps?

Agentic DevOps is the use of AI assistants or agents to support delivery work such as planning, coding, testing, security review, workflow troubleshooting, incident summaries, and release notes. In this bootcamp, AI support remains supervised by humans.

### Is this enough to get a job in AI or DevOps?

No single 30-day beginner bootcamp is enough by itself. This bootcamp builds the foundation needed to continue learning with better direction and confidence.

### Will I learn machine learning mathematics?

No. The bootcamp introduces AI and machine learning concepts at a practical beginner level. It does not cover advanced mathematics, model training theory, or production ML engineering.

### Will I deploy production cloud infrastructure?

No. The bootcamp introduces cloud and DevOps concepts but keeps hands-on work beginner-safe. Production deployment topics are better suited for a follow-up bootcamp.

## Participation

### Can teams join?

Yes. Learners may work individually or in small groups depending on the organizer's format. Every learner should still keep personal notes and make visible contributions.

### How should participants ask for help?

Participants should share:

- What they were trying to do
- What command or step they ran
- What happened
- What they expected to happen
- Any error message

### What is expected from mentors?

Mentors should guide beginners patiently, explain concepts clearly, encourage troubleshooting habits, and avoid turning the bootcamp into an advanced lecture series.

### What is the code of conduct?

Participants must follow MMAUG community standards. Be respectful, inclusive, patient, and constructive. Harassment, exclusionary behavior, or unsafe conduct is not acceptable.

## Projects and Submission

### Is there a final project?

Yes. Each participant should submit a small capstone project that combines AI and DevOps fundamentals.

### What should the capstone include?

The capstone should include:

- A GitHub repository
- A clear README
- One selected scenario track from the capstone expectations
- Source files or scripts
- Use of, or a clear beginner design for, Microsoft Foundry or Azure AI Services
- A basic automation workflow or validation checklist
- DevSecOps and AIOps notes
- A responsible AI note
- A short reflection

See [CAPSTONE-EXPECTATIONS.md](CAPSTONE-EXPECTATIONS.md) for the full requirements.

### What capstone project scenarios can I choose from?

Participants should choose one of three beginner-friendly scenario tracks:

- Learning Assistant
- Community FAQ Assistant
- Operations Helper

Each track should show how Microsoft Foundry or Azure AI Services can support the solution.

### Does the project have to be complex?

No. A simple, working, well-documented project is better than a complex unfinished project.

### Can I use sample data?

Yes. Use public, synthetic, or self-created sample data. Do not upload private, confidential, sensitive, or copyrighted data without permission.

### Can I submit a no-code project?

The bootcamp is beginner-friendly, but the final project should include at least one technical artifact such as a script, workflow, structured prompt library, documentation site, or automation file.

## Certificates and Recognition

### Will participants receive certificates?

This depends on MMAUG organizer policy. A certificate, badge, or showcase recognition can be offered if attendance, deliverables, and participation criteria are defined in advance.

### Are there prizes?

Yes. MMAUG plans to award 5 Microsoft Certification exam vouchers to the top 5 eligible participants whose final capstone submissions meet all required criteria.

### How can I qualify for a Microsoft Certification exam voucher?

To qualify for prize consideration, participants must submit by the stated deadline, meet the capstone criteria, use the official submission template, include Microsoft Foundry or Azure AI Services in the project or architecture, and provide evidence of AI, DevOps, DevSecOps, AIOps, and responsible AI work.

Full prize rules are available in [PRIZES.md](PRIZES.md).

### What evidence can participants show after the bootcamp?

Participants can share:

- Their GitHub bootcamp repository
- Daily notes
- Capstone project
- Final reflection
- Demo recording or screenshots if available
